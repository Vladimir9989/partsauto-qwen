# Загрузка проекта через FTP на Timeweb

## Шаг 1: Подготовка файлов для загрузки

1. Создайте ZIP-архив со всеми файлами проекта:
   - Выделите все файлы в папке `partsauto-site`
   - Создайте архив `partsauto-site.zip`
   - **Исключите из архива:**
     - `node_modules/` (слишком большой)
     - `.git/`
     - `dist/` (будет создан на сервере)

2. Или подготовьте список файлов для загрузки:
   ```
   partsauto-site/
   ├── package.json
   ├── package-lock.json
   ├── vite.config.js
   ├── ecosystem.config.js
   ├── .env.example
   ├── server/
   │   └── index.js
   ├── src/
   │   ├── App.jsx
   │   ├── index.css
   │   └── main.jsx
   ├── public/
   └── DEPLOY_TIMEWEB.md
   ```

## Шаг 2: Загрузка на Timeweb через FTP

1. Войдите в панель управления Timeweb
2. Перейдите в раздел "Файловый менеджер" или "FTP"
3. Создайте новую папку для проекта (например, `partsauto`)
4. Загрузите все файлы проекта в эту папку

## Шаг 3: Настройка Node.js приложения в Timeweb

1. В панели Timeweb найдите раздел "Node.js приложения"
2. Создайте новое приложение:
   - **Путь к приложению:** укажите папку с проектом (например, `partsauto`)
   - **Точка входа:** `server/index.js`
   - **Версия Node.js:** выберите последнюю LTS версию
   - **Порт:** оставьте по умолчанию (Timeweb сам назначит порт)

3. Настройте переменные окружения:
   - `NODE_ENV=production`
   - `PORT` (оставьте значение по умолчанию от Timeweb)
   - `XML_URL=https://ycf.partsauto.market/partsauto-feeds/avito_feeds/cb9901e562a4f410bd4f9bf80c21d094.xml`

## Шаг 4: Установка зависимостей

1. В панели Timeweb найдите кнопку "Установить зависимости" или используйте SSH
2. Если есть SSH доступ, подключитесь и выполните:
   ```bash
   cd /путь/к/partsauto
   npm install --production
   ```

## Шаг 5: Сборка фронтенда

1. Через SSH выполните:
   ```bash
   cd /путь/к/partsauto
   npm run build
   ```

2. Или если в Timeweb есть возможность запускать команды через панель, используйте её

## Шаг 6: Запуск приложения

1. В панели Timeweb нажмите "Запустить приложение"
2. Проверьте логи на наличие ошибок
3. Приложение будет доступно по временному URL от Timeweb

## Шаг 7: Настройка домена

1. В разделе "Домены" добавьте свой домен к приложению
2. Настройте DNS записи:
   - A-запись на IP сервера Timeweb
   - Или CNAME на предоставленный Timeweb адрес
3. Включите SSL сертификат (Let's Encrypt)

## Альтернативный вариант: Использование Git

Если Timeweb поддерживает Git:

1. Создайте репозиторий на GitHub/GitLab
2. В Timeweb настройте автоматическое развертывание из Git
3. Укажите команды для сборки:
   ```bash
   npm install --production
   npm run build
   ```

## Устранение проблем

**Приложение не запускается:**
- Проверьте логи в панели Timeweb
- Убедитесь, что точка входа указана правильно
- Проверьте, что все зависимости установлены

**Ошибки при сборке:**
- Убедитесь, что на сервере достаточно памяти
- Проверьте версию Node.js

**API не работает:**
- Проверьте, доступен ли XML URL с сервера Timeweb
- Проверьте настройки CORS
