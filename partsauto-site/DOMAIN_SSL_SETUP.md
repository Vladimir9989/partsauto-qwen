# Настройка домена и SSL на Timeweb

## Шаг 1: Добавление домена в панели Timeweb

1. Войдите в панель управления Timeweb
2. Перейдите в раздел "Домены и поддомены"
3. Нажмите "Добавить домен"
4. Введите ваш домен (например, `partsauto.ru`)
5. Выберите тип домена:
   - **Основной домен** - если это основной сайт
   - **Дополнительный домен** - если добавляете к существующему хостингу

## Шаг 2: Настройка DNS записей

### Вариант A: Домен зарегистрирован у Timeweb
Если домен куплен у Timeweb, DNS настраиваются автоматически.

### Вариант B: Домен зарегистрирован у другого регистратора
Нужно изменить DNS записи у вашего регистратора:

1. **Метод 1: Изменить NS-серверы** (рекомендуется)
   - Установите NS-серверы Timeweb:
     ```
     ns1.timeweb.ru
     ns2.timeweb.ru
     ns3.timeweb.org
     ns4.timeweb.org
     ```
   - Изменения вступят в силу через 24-48 часов

2. **Метод 2: Настроить A-запись и CNAME**
   - Создайте A-запись:
     ```
     @ (или пустое) → IP-адрес вашего сервера Timeweb
     ```
   - Создайте CNAME для www:
     ```
     www → ваш-домен.ru
     ```
   - IP-адрес сервера можно найти в панели Timeweb в разделе "Информация о сервере"

## Шаг 3: Привязка домена к приложению

1. В разделе Node.js приложений найдите ваше приложение
2. В настройках приложения найдите "Домены"
3. Добавьте ваш домен в список доменов приложения
4. Сохраните изменения

## Шаг 4: Настройка SSL сертификата (HTTPS)

Timeweb предоставляет бесплатные SSL сертификаты Let's Encrypt:

1. В разделе "Домены" найдите ваш домен
2. Нажмите "SSL сертификаты"
3. Выберите "Бесплатный SSL от Let's Encrypt"
4. Нажмите "Установить" или "Активировать"
5. Сертификат будет автоматически обновляться

### Альтернатива: Настройка вручную через SSH

Если нужно настроить SSL для Nginx:

1. Подключитесь по SSH к серверу
2. Установите certbot (если не установлен):
   ```bash
   sudo apt update
   sudo apt install certbot python3-certbot-nginx
   ```

3. Получите сертификат:
   ```bash
   sudo certbot --nginx -d ваш-домен.ru -d www.ваш-домен.ru
   ```

4. Настройте автоматическое обновление:
   ```bash
   sudo certbot renew --dry-run
   ```

## Шаг 5: Настройка редиректа с HTTP на HTTPS

### В панели Timeweb:
1. В настройках домена найдите "Перенаправления"
2. Создайте перенаправление:
   - С: `http://ваш-домен.ru`
   - На: `https://ваш-домен.ru`
   - Тип: 301 (постоянное)

### В Nginx конфигурации:
```nginx
server {
    listen 80;
    server_name ваш-домен.ru www.ваш-домен.ru;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name ваш-домен.ru www.ваш-домен.ru;
    
    ssl_certificate /etc/letsencrypt/live/ваш-домен.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/ваш-домен.ru/privkey.pem;
    
    # остальная конфигурация...
}
```

## Шаг 6: Настройка WWW и без WWW

Рекомендуется выбрать один основной вариант:

1. **Без WWW (example.ru)** - современный стандарт
2. **С WWW (www.example.ru)** - традиционный вариант

Настройте редирект с неосновного на основной вариант:

```nginx
# Редирект www на без www
server {
    server_name www.ваш-домен.ru;
    return 301 https://ваш-домен.ru$request_uri;
}
```

## Шаг 7: Проверка настройки

1. Проверьте DNS распространение:
   ```bash
   nslookup ваш-домен.ru
   dig ваш-домен.ru
   ```

2. Проверьте SSL сертификат:
   - Откройте `https://ваш-домен.ru` в браузере
   - Нажмите на значок замка → "Сертификат"
   - Убедитесь, что сертификат действителен

3. Проверьте редиректы:
   - `http://ваш-домен.ru` → `https://ваш-домен.ru`
   - `http://www.ваш-домен.ru` → `https://ваш-домен.ru`

## Шаг 8: Обновление приложения для работы с HTTPS

Убедитесь, что ваше Express приложение корректно работает с HTTPS:

1. В `server/index.js` добавьте поддержку проксированных заголовков:
   ```javascript
   app.set('trust proxy', 1);
   ```

2. Обновите CORS настройки если нужно:
   ```javascript
   app.use(cors({
     origin: ['https://ваш-домен.ru', 'https://www.ваш-домен.ru']
   }));
   ```

## Устранение проблем

**Домен не разрешается:**
- Подождите 24-48 часа для распространения DNS
- Проверьте DNS записи через whois или dig

**SSL сертификат не работает:**
- Убедитесь, что домен правильно разрешается на сервер
- Проверьте, открыт ли 443 порт в фаерволе

**Смешанный контент (Mixed Content):**
- Убедитесь, что все ресурсы (CSS, JS, изображения) загружаются по HTTPS
- Обновите XML_URL на HTTPS если нужно
