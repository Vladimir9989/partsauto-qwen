# Развертывание PartsAuto на Timeweb

## 📋 Краткое руководство

### Быстрый старт (за 10 минут)
1. **Соберите проект**: `npm run build`
2. **Создайте ZIP-архив** всех файлов (кроме `node_modules` и `.git`)
3. **Загрузите** в панель Timeweb через FTP
4. **Создайте Node.js приложение** в панели Timeweb
5. **Настройте переменные окружения**
6. **Запустите приложение**
7. **Добавьте домен** и **включите SSL**

## 🚀 Подробная инструкция

### 1. Подготовка проекта
```bash
cd partsauto-site
npm install
npm run build
```

### 2. Варианты развертывания на Timeweb

#### Вариант A: Node.js хостинг (проще всего)
1. В панели Timeweb → "Node.js приложения" → "Создать"
2. Загрузите файлы через FTP или файловый менеджер
3. Укажите точку входа: `server/index.js`
4. Установите зависимости через панель или SSH
5. Запустите приложение

#### Вариант B: VPS/VDS (полный контроль)
1. Подключитесь по SSH к серверу
2. Установите Node.js, npm, PM2
3. Скопируйте файлы проекта
4. Установите зависимости: `npm install --production`
5. Запустите с PM2: `pm2 start ecosystem.config.js`

#### Вариант C: Git развертывание
Если Timeweb поддерживает Git:
1. Создайте репозиторий
2. Настройте автоматическое развертывание
3. Укажите команды сборки в настройках

### 3. Настройка сервера для постоянной работы

#### С PM2 (рекомендуется)
```bash
npm install -g pm2
pm2 start ecosystem.config.js --env production
pm2 save
pm2 startup
```

#### Как systemd служба
Создайте файл `/etc/systemd/system/partsauto.service`:
```ini
[Unit]
Description=PartsAuto Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/partsauto
Environment=NODE_ENV=production
ExecStart=/usr/bin/node server/index.js
Restart=always

[Install]
WantedBy=multi-user.target
```

### 4. Настройка домена и SSL

#### В панели Timeweb:
1. "Домены" → "Добавить домен"
2. Укажите ваш домен
3. "SSL сертификаты" → "Бесплатный SSL от Let's Encrypt"
4. Настройте редирект HTTP → HTTPS

#### DNS настройки:
- **A-запись**: `@` → IP вашего сервера Timeweb
- **CNAME**: `www` → ваш-домен.ru
- Или используйте NS-серверы Timeweb

### 5. Проверка работоспособности

#### Базовые тесты:
```bash
# Проверка доступности
curl -I https://ваш-домен.ru

# Проверка API
curl "https://ваш-домен.ru/api/products?limit=1"

# Проверка health-check
curl "https://ваш-домен.ru/api/health"
```

#### Автоматический тест:
Запустите `test-deployment.sh` (из `TEST_DEPLOYMENT.md`)

### 6. Мониторинг и обслуживание

#### Логи:
```bash
# Логи приложения
pm2 logs partsauto-server

# Логи Nginx
tail -f /var/log/nginx/access.log
```

#### Мониторинг ресурсов:
```bash
# Использование памяти
pm2 monit

# Статус процессов
pm2 status
```

#### Автоматическое обновление:
```bash
# Обновление зависимостей
npm update

# Перезапуск при изменениях
pm2 restart partsauto-server
```

## 🛠️ Устранение неполадок

### Проблема: Сервер не запускается
**Решение:**
1. Проверьте логи: `pm2 logs partsauto-server`
2. Убедитесь, что порт не занят
3. Проверьте переменные окружения

### Проблема: API не возвращает данные
**Решение:**
1. Проверьте доступность XML URL с сервера
2. Проверьте кэширование в `server/index.js`
3. Увеличьте `CACHE_DURATION` если нужно

### Проблема: Высокая нагрузка на сервер
**Решение:**
1. Увеличьте кэширование
2. Настройте пагинацию (уже есть)
3. Рассмотрите использование CDN для статики
4. Увеличьте ресурсы сервера в панели Timeweb

### Проблема: SSL сертификат не работает
**Решение:**
1. Проверьте, что домен правильно разрешается
2. Обновите сертификат в панели Timeweb
3. Проверьте настройки Nginx/Apache

## 📈 Оптимизация производительности

### Для Timeweb Node.js хостинга:
1. Включите кэширование статики
2. Используйте CDN для изображений
3. Оптимизируйте размер сборки Vite
4. Настройте сжатие Gzip

### Для VPS:
1. Настройте Nginx как обратный прокси
2. Включите кэширование в Nginx
3. Используйте Redis для кэширования API
4. Настройте балансировку нагрузки если нужно

## 🔄 Обновление приложения

### Процесс обновления:
1. Остановите приложение: `pm2 stop partsauto-server`
2. Загрузите новые файлы
3. Установите обновленные зависимости: `npm install`
4. Пересоберите фронтенд: `npm run build`
5. Запустите приложение: `pm2 start partsauto-server`

### Бесперебойное обновление (zero-downtime):
```bash
pm2 reload partsauto-server
```
